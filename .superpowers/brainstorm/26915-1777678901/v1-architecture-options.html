Seedr Clone V1 Architecture Options

Pick the deployment shape we should use for the first build.

A

Recommended: Single Leaseweb Docker Server

Laravel/Inertia, Postgres, Redis, qBittorrent-nox, RustFS, and reverse proxy run on one 10Gbps Leaseweb server.

Pros

  • Fastest to build and deploy.
  • Simple networking between services.
  • Works well for a controlled v1 with 700MB quotas.

Cons

  • One server is a single point of failure.
  • Scaling later requires splitting services.
B

Two-Server Split

Laravel/Postgres/Redis on one server, qBittorrent and RustFS on another storage-heavy server.

Pros

  • Cleaner isolation between app and storage/download load.
  • Easier to upgrade storage separately.

Cons

  • More expensive and more moving parts.
  • Private networking, firewalls, and backups are more complex.
C

Kubernetes / Cluster

Run app, workers, storage, and torrent engine across a managed or self-hosted cluster.

Pros

  • Best long-term scaling path.
  • Better service isolation and automation.

Cons

  • Too much operational weight for v1.
  • Slows product development before the core workflow is proven.
